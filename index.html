Agent architecture

EBR Generator — two-model design, RAG pipeline, eval loop

EBR Generator Agent — end-to-end flow
Haiku routes · Sonnet generates · Glean index retrieves · human reviews at-risk outputs

A small, fast model handles classification cheaply. A large model only fires when generation is needed. Retrieval happens between routing and generation — context is injected at query time, never baked into the model. This is how you keep cost-per-account manageable at 10 accounts today and 100 tomorrow.
