EBR Generator — agent architecture
Two-model design · RAG pipeline · eval loop

Uses a small router model for classification (cheap, fast) and a larger generation model only when needed. Retrieval happens between routing and generation — context is injected at query time, never baked into the model.

Trigger layer
Scheduled trigger (Mon 7am)
→
Pull account signals (usage, health, last exec touch)
Router layer — small model (Haiku)
Classify: at-risk / healthy / expansion
→
Select prompt template + retrieval scope
Low-cost classification only — no generation. Routes to the correct retrieval strategy without full model cost. ~70% of cost savings happens here.
Retrieval layer (RAG via Glean index)
Query Glean index: case studies + account history
→
Score + filter (similarity threshold: 0.75)
→
Inject into context window
Fallback: if no chunk scores above 0.75 → insert [CONTEXT PLACEHOLDER] and flag for human review. Never allow the model to fill the gap with parametric memory.
Generation layer — large model (Sonnet, temp: 0.3)
System prompt + retrieved context + account signals
→
Generate structured EBR draft
Guardrails enforced in system prompt: no fabricated metrics, cite sources, escalate if renewal <90d, output schema via JSON mode.
Output + eval layer
Schema validation
→
Human review queue (at-risk only)
→
Deliver to AOM inbox
Eval: weekly golden set of 15 accounts · scored on factual accuracy, format compliance, guardrail adherence · regression test before any prompt update ships.
